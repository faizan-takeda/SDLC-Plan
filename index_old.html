07 / The question that decides the vendor

Name one capability the vendor delivers that GitHub or Anthropic won’t ship natively within 12 months.

…and show the validation burden it removes — not relocates.

No clear answer → no vendor.

08 / The Split

The thread splits three ways.

Adopt — don’t build

The commodity layer

  • Code generation
  • Environment creation
  • Agent orchestration — GitHub already runs issue-to-PR agents

It’s a subscription. It improves monthly without Takeda spending a person-hour.

Build — only this

The GxP layer

  • Validation artifacts generated with the code
  • Audit trails an inspector can walk
  • Traceability — requirement → test → release
  • Agent guardrails for regulated repos

No vendor ships Takeda’s compliance context. This is the differentiated work.

Skip

The distractions

  • Monorepo migration
  • Custom code-gen platform
  • Custom orchestration layer over interactive coding

Solving problems the platform already solved — or nobody has.

09 / The Pathway

90 days. Data before decisions.

No vendor decision until the baseline exists — which is not “no”, it’s “not before evidence.” Nobody votes against evidence.

You are here — Week 0
Weeks 1–4

Baseline

Pull GitHub metrics: how many developers, how much agent use, how much AI code merges. Answers “do we even need this — and where.”

Weeks 3–6

Adopt / Build / Skip map

Publish the split, checked against the baseline — not opinions.

Weeks 5–8

GxP prototype

Thin slice on 1–2 regulated repos: artifacts, trail, traceability.

Weeks 9–12

Decide with data

Return with numbers. Vendor scope: likely none — or narrow.

Run by 5–8 people. Not 100. Not a vendor.
